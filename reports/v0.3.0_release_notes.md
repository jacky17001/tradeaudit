# TradeAudit v0.3.0 Release Notes

Date: 2026-04-07
Release Type: Stage22-27 consolidated pre-release

## Highlights (Stage22-27)

### Stage22 - Candidate tagging foundation
- Added candidate mark/unmark flow for strategies.
- Added candidate-only filtering on Backtests list.
- Persisted candidate status in DB and API responses.

### Stage23 - Forward Runs registry
- Added Forward Run creation from candidate strategies.
- Added Forward Run list with status filter and pagination.
- Added strict status transition API (RUNNING/PAUSED/COMPLETED/FAILED).

### Stage24 - Forward summary capture
- Added create/edit/get Forward Summary per run.
- Embedded summary data into run list payloads.
- Added backend repository/service/API coverage and smoke validation.

### Stage25 - Gate result capture
- Added create/edit/get Gate Result per run.
- Added Gate Results list with decision filter and pagination.
- Embedded gate result into run list payloads.

### Stage26 - Strategy lifecycle view
- Added unified lifecycle API: backtests + candidate + source job + latest run + latest summary + latest gate.
- Added Backtests lifecycle modal and per-strategy lifecycle action.
- Added route/service tests and stage smoke for end-to-end lifecycle aggregation.

### Stage27 - v0.3.0 release polish
- Refined Backtests import wording and added clear import guidance/help block.
- Unified Forward/Gate/Lifecycle loading/error/empty messaging for consistent UX.
- Applied small UI consistency fixes (labels, close actions, section readability).
- No major feature expansion, no architecture refactor.
